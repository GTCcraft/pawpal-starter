# PawPal+ Project Reflection

## 1. System Design

**a. Initial design**
Given these core requirements:
1. A user should be able to see the daily plan generated by the app;
2. A user should be able to add pets;
3. A user should be able to add tasks, with each task corresponding to a pet. 

The necessary classes are:
1. An owner class. Owners have a name and a list of pets. They are able to add pets.
2. A pet class. Pets have names, owners, and tasks associated with them. They are able to add tasks to themselves: add_task(). Also get_tasks().
3. A task class. Each task has a name and is associated with one pet. Also has a priority level (int) and duration in min. 
4. A scheduler class. Schedulers take a list of tasks and run an algorithm to generate a daily plan. plan() method takes an owner and creates a daily plan for that owner; also has a helper method to add tasks to a given daily plan.
5. A daily plan class. display() shows the plan. 

**b. Design changes**
Made task priority take an enum + created Priority enum w/ constant strings to make code more readable. 

Also made Scheduler's plan() method take a Pet, not an Owner, since each schedule will be for one pet only.
